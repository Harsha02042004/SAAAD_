Frequently Asked Questions

Learn more about the design, analysis, and features of the SAAAD - Sialic Acid Analogs ADME Prediction Database.

SAAAD is a computational database of novel sialic acid derivatives designed and analyzed using cheminformatics workflows. It integrates molecular design, structural analysis, and drug-likeness evaluation into a single interactive platform.

No. All compounds in this database are internally designed as part of a computational drug discovery workflow. They are not directly sourced from external repositories such as PubChem.

The compounds were generated through structural modification of sialic acid scaffolds, followed by computational processing including:
  • SMILES-based molecular design
  • 3D conformer generation (RDKit)
  • Geometry optimization
  • Descriptor calculation

Each compound includes:
  • 2D and 3D molecular structures
  • SMILES, InChI, molecular formula
  • Physicochemical descriptors (MW, LogP, TPSA, etc.)
  • Drug-likeness rule evaluations
  • Composite scoring and ranking
  • Structural fingerprints

Drug-likeness is evaluated using multiple rule-based filters:
  • Lipinski Rule of Five
  • Veber Rule
  • Ghose Filter
  • Egan Rule
  • Muegge Criteria
A penalty-based scoring system is used to rank compounds.

Principal Component Analysis (PCA) reduces high-dimensional descriptor data into a 2D representation, allowing users to explore chemical space, identify clusters, and compare compounds visually.

Molecular similarity is calculated using fingerprint-based methods and Tanimoto similarity, enabling identification of structurally related compounds and nearest neighbors.

The platform is intended for:
  • Drug discovery researchers
  • Medicinal chemists
  • Bioinformatics and cheminformatics students
